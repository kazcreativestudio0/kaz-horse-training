# 画像の優先順位と反映方法

## 📋 プラン画像の優先順位（上から順）

### 1. **plans/ フォルダ内でキーワードを含むファイル名**
ファイル名に以下のキーワードを含む画像が優先的に使用されます：

| プラン | キーワード | 例 |
|--------|-----------|-----|
| 体験乗馬（ショート） | `trial` | `trial.jpg`, `plan-trial.jpg`, `IMG_trial_4965.jpg` |
| 体験乗馬（ロング） | `horse-care` または `horsecare` | `plan-horse-care.jpg`, `horse-care-image.jpg` |
| レッスン回数券 | `lesson` | `plan-lesson.jpg`, `lesson-photo.jpg` |

**重要**: ファイル名にキーワードを含めれば、どのような名前でもOKです！
- ✅ `trial-photo.jpg` → 体験乗馬（ショート）に使用
- ✅ `IMG_4965_trial.JPG` → 体験乗馬（ショート）に使用
- ❌ `IMG_4965.JPG` → キーワードがないので、次の優先順位へ

### 2. **plans/ フォルダ内の最初の画像（ファイル名順）**
キーワードが見つからない場合、フォルダ内の画像をファイル名順に並べて、最初の画像を使用します。

### 3. **ルートフォルダ（assets/local-images/）の画像**
`plan-trial.*`, `plan-horse-care.*`, `plan-lesson.*` という名前のファイルを検索します。

### 4. **Google DriveのフォールバックURL**
上記すべてが見つからない場合のみ、Google DriveのURLを使用します。

## 🔧 画像を反映させる方法

### 方法1: ファイル名にキーワードを含める（推奨）

現在のファイル：
- `IMG_4965.JPG` → 体験乗馬（ショート）用にしたい場合
- `IMG_4968.JPG` → レッスン回数券用にしたい場合

**リネーム例：**
```bash
# 体験乗馬（ショート）用
IMG_4965.JPG → plan-trial.jpg
# または
IMG_4965.JPG → trial-photo.jpg
# または
IMG_4965.JPG → IMG_4965_trial.JPG

# レッスン回数券用
IMG_4968.JPG → plan-lesson.jpg
# または
IMG_4968.JPG → lesson-image.jpg
```

### 方法2: 開発サーバーを再起動

画像ファイルを追加・変更した場合、Viteの開発サーバーを再起動してください：

1. ターミナルで `Ctrl + C` を押してサーバーを停止
2. `npm run dev` で再起動
3. ブラウザをリロード（`Cmd + R` または `F5`）

## 🎯 推奨されるファイル名

以下のようなファイル名にすると、どのプランに使われるかが明確になります：

```
assets/local-images/plans/
├── plan-trial.jpg        ← 体験乗馬（ショート）
├── plan-horse-care.jpg   ← 体験乗馬（ロング） ✅ 既に存在
└── plan-lesson.jpg       ← レッスン回数券
```

または、日本語を含めてもOKです：

```
assets/local-images/plans/
├── 体験乗馬ショート.jpg
├── 体験乗馬ロング.jpg
└── レッスン回数券.jpg
```

ただし、キーワード（`trial`, `horse-care`, `lesson`）を含める必要があります。

## ⚠️ よくある問題と解決方法

### 問題1: 画像が反映されない

**原因**: 
- ファイル名にキーワードが含まれていない
- 開発サーバーが再起動されていない

**解決方法**:
1. ファイル名を確認してキーワードを含める
2. 開発サーバーを再起動（`Ctrl + C` → `npm run dev`）
3. ブラウザのキャッシュをクリア（`Cmd + Shift + R`）

### 問題2: プレースホルダー（SVG）が表示されている

**原因**: 
- SVGファイルにキーワードが含まれているため、そちらが優先されている

**解決方法**:
1. SVGファイルを削除するか、`placeholder/` フォルダに移動
2. JPG/PNGファイル名にキーワードを含める

## 📝 まとめ

1. **ファイル名にキーワードを含める** → 確実に反映される
2. **開発サーバーを再起動** → 新しいファイルを認識
3. **ブラウザをリロード** → 変更を確認

最も簡単な方法は、`IMG_4965.JPG` → `plan-trial.jpg` のようにリネームすることです！
