# 画像フォルダ構造ガイド

このフォルダは、サイトで使用するすべての画像を整理するためのフォルダです。
各フォルダに画像を入れるだけで、自動的にサイトに反映されます。

## 📁 フォルダ一覧

### `hero/` - ヒーロー背景画像
- **用途**: トップページの全画面背景画像
- **推奨サイズ**: 横長（16:9）、幅1920px以上
- このフォルダに入れた最初の画像が使用されます

### `logo/` - ロゴ画像
- **用途**: サイト全体で使用されるロゴ（ヘッダー、Hero、フッター）
- **推奨サイズ**: 正方形または横長、幅300px〜500px
- 背景が透明なPNGまたはSVG形式を推奨

### `about/` - Aboutセクション画像
- **用途**: Aboutセクションのメイン画像
- **推奨サイズ**: 縦長（3:4）、幅800px〜1200px
- このフォルダに入れた最初の画像が使用されます

### `horses/` - 馬の紹介画像
- **用途**: 馬の紹介セクション（メイン3頭 + ギャラリー）
- **推奨サイズ**: 横長（4:3）、幅800px〜1200px
- 画像はファイル名順に自動的に並びます
- 最初の3枚がメイン3頭として使用され、残りがギャラリーに表示されます

### `plans/` - 料金プラン画像
- **用途**: 料金プランセクションの各プランカード
- **推奨サイズ**: 横長（16:9または4:3）、幅800px〜1200px
- ファイル名に以下を含めると、自動的に該当プランに割り当てられます：
  - `trial` → 体験乗馬（ショート）
  - `horse-care` または `horsecare` → 体験乗馬（ロング）
  - `lesson` → レッスン回数券

### `gallery/` - ギャラリー画像
- **用途**: Aboutセクションの下部ギャラリー
- **推奨サイズ**: 正方形（1:1）または縦長（4:3）
- このフォルダに入れたすべての画像が自動的に表示されます
- 画像はファイル名順に並びます

### `maps/` - 地図画像
- **用途**: Contactセクションの地図エリア
- **推奨サイズ**: 横長（16:9または4:3）、幅800px〜1200px
- このフォルダに入れた最初の画像が使用されます

### `placeholder/` - プレースホルダー（未使用時）
- 一時的に使用されていないプレースホルダー画像を保存するフォルダです
- サイトには表示されません

## 🎯 使い方

1. 各フォルダに画像ファイルをコピーするだけです
2. ファイル名は基本的に自由です（`plans/`フォルダのみ、ファイル名で識別されます）
3. 対応形式：`.png`, `.jpg`, `.jpeg`, `.webp`, `.svg`, `.avif`, `.JPG`, `.PNG`
4. 画像を追加・削除すると、自動的にサイトに反映されます（開発サーバーの再起動が必要な場合があります）

## 📝 注意事項

- 各フォルダの詳細な使い方は、フォルダ内の `README.md` を参照してください
- 画像ファイルが存在しない場合、Google DriveのフォールバックURLが使用されることがあります
- 画像の最適化（ファイルサイズの削減）を推奨します

## 🔄 更新の反映

画像を追加・変更・削除した後は：
1. 開発サーバーが起動中の場合、ブラウザをリロード（自動反映される場合があります）
2. 反映されない場合は、開発サーバーを再起動してください（`npm run dev`）
